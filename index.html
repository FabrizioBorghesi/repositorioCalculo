Comparação de eficiência dos algoritmos

Aqui uma tabela de comparação onde foram testados arrays (100, 300, 400, 500, 700, 800, 900, 1000) e seus tempos de execução em segundos registrados.

Sobre a eficiência

Bubble Sort. O tempo cresce a medida que os arrys crescem, então para grandes arrays não é muito viável.
Selection Sort. Segue a mesma lógica que o Bubble Sort, mesmos endo um pouco melhor.
Insertion Sort. Possui o melhor desempenho de listas parcialmente ordenadas, mas continua sendo ineficiente para listas grandes ordenadas.
Merge Sort. Mais eficiente sendo ideal para grandes conjuntos de dados.

Conclusão

Bubble Sort e Selection Sort são ótimas opções para listas muito pequenas por serem ineficientes quando se trata de listas muito grandes. Porém, para listas pequenas ou quase ordenadas é uma ótima opção, enquanto a Merge Sort é a escolha ideal para grandes listas ou desordenadas.

Merge Sort

Algoritmo Merge Sort

O Merge Sort é um algoritmo de ordenação eficiente, baseado na técnica de divisão e conquista. Isso é, sua abordagem recursiva ordena os dados separando-os em duas metades, ordenando cada uma recursivamente e depois mesclando as 2 partes para ordená-las e produzir o array final ordenado.

Em ordem, o algoritmo segue os seguintes passos:

1. Divide todo o array em 2 metades.
2. Ordena cada metade recursivamente.
3. Mescla as 2 metades para produzir o array final.

Bubble Sort

Algoritmo Bubble Sort

O Bubble Sort é um algoritmo de ordenação simples que funciona percorrendo repetidamente a lista a ser ordenada, comparando os pares de elementos adjacentes, trocando a posição se estiverem na ordem errada. O processo se repete até que nenhuma troca seja mais necessária, indicando que a lista está ordenada.

Em ordem, o algoritmo segue os seguintes passos:

1. Percorre por toda lista.
2. Compara os elementos adjacentes, trocando-os caso estejam na ordem errada.
3. Realiza uma repetição até que todos os elementos estejam na ordem correta.

Insertion Sort

Algoritmo Insertion Sort

O algoritmo de ordenação Insertion Sort funciona percorrendo um array, inserindo cada elemento na posição correta em relação aos demais ordenados, expandindo a porção de arrays.

Em ordem, o algoritmo segue os seguintes passos:

1. Percorre por todo array.
2. A cada elemento, ele compara com o elemento à esquerda em sua porção do array e insere na posição correta.

Busca Binária

Algoritmo Busca Binária

A busca binária é um algoritmo que funciona para encontrar um elemento em um array ordenado. Ele compara o elemento buscado com o elemento do meio do array e, dependendo da comparação, continua a busca na metade esquerda ou direita do array, reduzindo o intervalo de busca pela metade a cada iteração até localizar.

Em ordem, o algoritmo segue os seguintes passos:

1. Inicia os limites de intervalos da busca.
2. Enquanto o intervalo não estiver vazio, calcula o elemento do meio do array comparando-o com o elemento buscado, ajustando os limites de busca.
3. Retorna o índice do elemento caso for encontrado, senão indica que o elemento é inexistente no array.

Selection Sort

Algoritmo Selection Sort

O algoritmo de ordenação Selection Sort é um algoritmo simples que funciona dividindo o array em duas partes: a parte ordenada na frente e a parte não ordenada atrás. Inicialmente, a parte ordenada está vazia e a parte desordenada contém todos os elementos. Isso é, em cada iteração, o algoritmo busca o menor elemento desordenado e o realoca na parte ordenada.

Em ordem, o algoritmo segue os seguintes passos:

1. Percorre por todo array
2. Encontra o menor elemento na parte não ordenada.
3. Realiza a troca do menor elemento encontrado com o primeiro elemento não ordenado.
4. Realiza uma repetição até que todo array esteja ordenado.

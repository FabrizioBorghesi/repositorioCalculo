Comparação de eficiência dos algoritmos

Aqui uma tabela de comparação onde foram testados arrays (100, 300, 400, 500, 700, 800, 900, 1000) e seus tempos de execução em segundos registrados.

Sobre a eficiência

Bubble Sort. O tempo cresce a medida que os arrys crescem, então para grandes arrays não é muito viável.
Selection Sort. Segue a mesma lógica que o Bubble Sort, mesmos endo um pouco melhor.
Insertion Sort. Possui o melhor desempenho de listas parcialmente ordenadas, mas continua sendo ineficiente para listas grandes ordenadas.
Merge Sort. Mais eficiente sendo ideal para grandes conjuntos de dados.

Conclusão

Bubble Sort e Selection Sort são ótimas opções para listas muito pequenas por serem ineficientes quando se trata de listas muito grandes. Porém, para listas pequenas ou quase ordenadas é uma ótima opção, enquanto a Merge Sort é a escolha ideal para grandes listas ou desordenadas.
